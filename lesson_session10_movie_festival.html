The problem in plain words

The problem in plain words

A festival is showing n movies, one screen, no overlaps allowed. Each movie has a start time and an end time. You want to watch as many whole movies as possible. What's the biggest number of movies you can fit, back to back, with no two overlapping?

This is a different shape of problem from anything in Stage 3 so far. It isn't two pointers closing inward on a sorted list — it's a brand new idea called greedy interval scheduling: at each step, make the choice that looks best right now, and trust that it stays best all the way to the end.

The obvious idea — and why it fails

The natural first guess: watch whichever movie starts earliest. That sounds reasonable, but it can trap you.

A long movie that starts early can block out several short ones later — "starts earliest" doesn't protect you from that.

The fix: sort by when movies END

What actually matters isn't when a movie starts — it's how much time it leaves free afterward. A movie that ends early leaves the rest of the day wide open, no matter when it started. So: sort every movie by its end time, smallest first, and walk through them in that order — taking a movie whenever it starts at or after the end of the last movie you kept.

Sorted by end time, the greedy walk naturally prefers movies that leave more room behind them.

Common mistakes on this one

✕
Sorting by start time instead of end time — the intuitive guess, and the one that fails on cases like the one above.
✕
Sorting by how long each movie is (shortest first) — another common guess, also wrong: a short movie in the wrong spot can still block out more than it saves.
✕
Updating the "last end" tracker on every movie you look at, instead of only the ones you actually decide to keep.
✕
Using > instead of >= when checking if a movie fits — a movie that starts exactly when the last one ends is allowed.

Hint 1

Hint 2
Try your current code on this by hand: Movie A runs 0 to 10. Movies B and C run 1 to 2 and 3 to 4. If you pick whichever starts earliest, what happens to B and C? What would you need to sort by so A doesn't get picked before them?
Hint 3

A trick just for this one

If your count looks wrong, print your sorted movies list right after sorting, before the loop. The end times (second number in each pair) should be going up the whole way down the list. If they're not, you sorted by the wrong thing.
