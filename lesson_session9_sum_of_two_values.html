Sum of Two Values — Pairing Positions Before You Sort

Last time: Ferris Wheel — two pointers starting from opposite ends Today: the same two-pointer trick, plus a fix for a trap it hides

The problem in plain words

The problem in plain words

You get a list of numbers and a target number x. Find two numbers in the list — at two different spots — that add up to exactly x. Print which two spots they were at (position 1, 2, 3...), not the numbers themselves. If no two numbers work, print IMPOSSIBLE.

This sounds almost identical to Ferris Wheel — sort, then close in from both ends. It mostly is. But there's one new trap, and it's the whole point of today's session.

The trap: sorting throws away the positions

In Ferris Wheel, you only ever needed to count gondolas — you never had to say which kid was which afterward. Today you do. The problem wants the original positions, from before you sorted anything. And list.sort() doesn't care about that at all — it just rearranges the numbers and moves on.

Once you sort a plain list of values, there's no way back to "which original spot was this."

The fix: tag each value with its position before you sort

Instead of sorting a list of plain numbers, build a list of (value, position) pairs first — then sort that. Python sorts tuples by their first item by default, so sorting a list of (value, position) pairs sorts by value, exactly like before, except now the position rides along for the trip.

Wrong — sorts the values, loses the positions
n, x = map(int, input().split()) values = list(map(int, input().split())) values.sort() left = 0 right = n - 1 while left < right: total = values[left] + values[right] if total == x: print(left + 1, right + 1) # these are SORTED positions, break # not the original ones! elif total < x: left += 1 else: right -= 1 else: print("IMPOSSIBLE")
Right — pair with position first, sort the pairs
n, x = map(int, input().split()) values = list(map(int, input().split())) pairs = [] for i in range(n): pairs.append((values[i], i + 1)) # (value, original 1-indexed position) pairs.sort() # sorts by value, position rides along left = 0 right = n - 1 while left < right: total = pairs[left][0] + pairs[right][0] if total == x: print(pairs[left][1], pairs[right][1]) # ORIGINAL positions break elif total < x: left += 1 else: right -= 1 else: print("IMPOSSIBLE")
Same sort, same order of values — but every value still carries the position it started at.

Walking the real CSES sample

Array 2 7 5 1, target x = 8. Same two-pointer idea as Ferris Wheel — pointers start at opposite ends of the sorted pairs and close in — except now you're checking sums against x instead of counting gondolas.

Values 7 (position 2) and 1 (position 4) sum to 8. Either order you print them in is correct.

A trick just for this one

If your two positions look wrong, print your pairs list right after building it, before you sort. Each entry should look like (value, position). If it's just plain numbers with no position attached, that's the bug.
