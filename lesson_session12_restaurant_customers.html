Restaurant Customers — Counting Overlaps Instead of Picking Movies

Open Restaurant Customers on CSES ↗

The problem in plain words

The problem in plain words

You get n customers, each with an arrival time and a leaving time. What's the largest number of customers who were in the restaurant at the same moment, at any point in the day?

This isn't about picking which customers to keep, the way Movie Festival was about picking which movies to watch. Every customer already showed up — you just need to find the busiest single moment. That's a different question, and it needs a different shape of sorting.

The trap: sorting arrivals and departures together, the wrong way

A tempting shortcut: mark every arrival as "+1", every departure as "-1", combine them all into one list of (time, label) pairs, sort the whole thing, and walk through counting up and down. It mostly works — until an arrival and a departure land at the exact same time, and the sort has to decide which one goes first.

When an arrival and a departure land at the same instant, the departure has to be counted first, or overlaps that never really happened get counted anyway.

The fix: two separately-sorted lists, walked with two pointers

Instead of one combined list with a tricky tie-break, build two lists — all arrival times, and all leaving times — and sort each one on its own. Then walk through both at once with two pointers, the same shape you've used since Apartments: at each step, compare the next arrival to the next departure. Whichever is smaller happened first.

Comparing the two lists directly settles the tie the right way — a departure at the same instant as an arrival is processed first.
Wrong — one combined list, sorted with a backwards tie-break
n = int(input()) events = [] for _ in range(n): a, b = map(int, input().split()) events.append((a, "in")) events.append((b, "out")) events.sort() # ties break alphabetically: "in" before "out" -- backwards! count = 0 best = 0 for time, kind in events: if kind == "in": count += 1 best = max(best, count) else: count -= 1 print(best)
Right — two lists, sorted and compared separately

Why this stops short on purpose

The new idea today is splitting one combined list into two separate ones — the two-pointer walk itself is the same shape you already wrote for Apartments. Paste just the right-hand block on its own and run it: it builds and sorts both lists, then stops, printing nothing. That silence means the two-pointer walk is still left for you to write.

Common mistakes on this one

✕
Combining arrivals and departures into one list and letting a simple sort decide ties — strings like "in"/"out" sort alphabetically, which is backwards from what this problem needs.
✕
Using <= instead of < when comparing the next arrival to the next departure — that single symbol decides which one wins a tie.
✕
Forgetting to check for a new best count after every single arrival, not just at the end.
✕
Advancing both pointers on the same step, instead of moving only the one that matched.

Looking back at this run

Three sessions on one technique family

Movie Festival, Movie Festival II, and Restaurant Customers all come from the same idea: sort something sensibly, then scan through it once, making a simple decision at each step. That's the same rhythm two pointers had across Apartments, Ferris Wheel, and Sum of Two Values. If this run felt smoother by the third session than the first, that's the pattern to notice — it's the same thing that happened with two pointers, and it's a good sign you're generalizing the underlying idea, not just memorizing three separate solutions.

A trick just for this one
